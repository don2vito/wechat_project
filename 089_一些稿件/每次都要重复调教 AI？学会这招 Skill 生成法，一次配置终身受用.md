你有没有过这样的经历？让 AI 帮你写一篇文章，它写得乱七八糟；让它帮你处理表格，它总是搞错格式；让它帮你写代码，bug 多到你想砸电脑？
这不是 AI 不够聪明，而是它不知道你的做事方法。就像一个刚入职的新员工，虽然很聪明，但不知道公司的流程、规范和标准，所以总是做不好事情。
Skill 就是专门解决这个问题的 "员工培训手册"，它能把你的经验、流程和最佳实践打包成一个文件，让 AI 瞬间从 "职场小白" 变成 "职场老司机"。

## 一、Skill 的构成

Skill 就是 AI 的 SOP 手册。它是一个打包好的文件夹，里面包含了完成特定任务的所有指令、流程、模板和参考资料。当 AI 需要完成这个任务时，它会自动打开这个手册，按照里面的步骤一步步执行。
那 Skill 不就是长一点的提示词吗？其实不完全一样，就像 "口头吩咐" 和 "标准操作手册" 的区别。

| 普通提示词 (Prompt)  | Skill (技能包)             |
| -------------------- | -------------------------- |
| 一次性的，用完就忘   | 永久保存，随时调用         |
| 只能解决一个具体问题 | 能解决一类问题             |
| 需要每次都写很长     | 一次编写，终身受益         |
| 只能包含文字指令     | 可以包含代码、模板、文件等 |
| 不能自动触发         | 可以根据语义自动触发       |

一个标准的 Skill 其实就是一个文件夹，里面包含以下几个部分。其中最重要的就是 **SKILL.md** 文件，它就像技能的 "说明书"，告诉 AI 这个技能能做什么、什么时候用、怎么用。

```plaintext
my-skill/
├── SKILL.md  # 核心文件，技能的说明书
├── scripts/  # 可选，辅助脚本
├── templates/  # 可选，模板文件
└── references/  # 可选，参考资料
```

> 编写高质量 SKILL.md 的黄金法则：
>
> 1. **明确边界**：一定要写清楚"何时使用"和"不要使用"，这是避免AI错误调用的关键
> 2. **步骤具体**：执行步骤要细到AI能直接照着做，不要写"分析数据"这种模糊的话
> 3. **模板详细**：输出模板越详细，生成的结果就越符合你的预期
> 4. **错误处理**：提前考虑各种异常情况，让技能更健壮
> 5. **示例丰富**：多给几个使用示例，用户一看就知道怎么用

一个好的 SKILL.md 文件应该包含以下几个部分。**关键是要写好 description 字段**，它是最容易被忽略但最重要的部分，写的时候想象用户会怎么向 AI 提问，把那些关键词都放进去，AI 才能在正确的时候自动触发你的技能。

```plaintext
---
name: your-skill-name
description: >
  一句话说明这个技能做什么。
  当用户提到"关键词A"、"关键词B"或需要完成某类任务时，自动触发此技能。
  （这段描述的质量直接决定 AI 能否准确调用你的技能，请务必写好触发关键词。）
---

## 目标（Goal）

**不适用场景：**

- 如果需要读取参考文档：参考 [references/xxx.md](references/xxx.md)
- 如果需要使用模板：参考 [assets/template.md](assets/template.md)

### 第二步：[步骤名称]

具体说明……

### 第三步：[步骤名称]

具体说明……

## 输出要求（Output Rules）

- 输出格式：Markdown / Excel / PPT / 其他
- 必须包含的内容：[列出不可遗漏的要素]
- 质量标准：[描述合格输出的标准]

## 异常处理（Error Handling）

- 如果输入文件格式不对 → 提示用户："请上传 .xlsx 格式的文件"
- 如果数据为空 → 提示用户："未检测到有效数据，请检查文件内容"
- 如果执行脚本失败 → 检查依赖是否安装，提示具体错误信息
```

## 二、自动归纳流程生成 Skill 的方法

Anthropic 官方提供了一个神奇的工具叫 **Skill-Creator**，它本身就是一个 Skill，你只要告诉它想做什么，它就会引导一步步创建一个完整的 Skill，下载地址在 `https://github.com/anthropics/skills/tree/main/skills/skill-creator`。

> 使用步骤：
>
> 1. 安装 Skill-Creator
> 2. 告诉 AI ："我想做……事情"
> 3. AI 会问一系列问题：
>    - 这个 Skill 叫什么名字？
>    - 它能做什么？
>    - 什么时候应该使用它？
>    - 执行这个任务的步骤是什么？
>    - 输出格式有什么要求？
> 4. 回答这些问题，AI 就会自动生成一个完整的 SKILL.md 文件
> 5. 测试一下，如果有不满意的地方，告诉 AI 修改，它会自动调整
