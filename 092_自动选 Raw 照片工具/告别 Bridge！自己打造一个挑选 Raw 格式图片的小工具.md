# 告别 Bridge！自己打造一个挑选 Raw 格式图片的小工具

## 一、需求场景

喜欢拍照的朋友都会遇到这样的场景，在相机里设置 Jpeg + Raw 的双格式，文件名是一样的，后缀名不同。Jpeg 格式用来预览图片，Raw 格式用来后期修图。

每一次拍完照，都会有几百张照片，需要在这么多照片中查看、挑选，再把需要的 Raw 格式另存出来，来回操作费时费力，效率很低。

以上步骤固然可以通过 Adobe Bridge、Light Room、Capture One、DXO PhotoLab 等专业工具来处理，但也有一个问题，软件本身很臃肿，启动慢、占资源，最坏的是需要额外划出一大块硬盘空间来做索引和缓存。对我来说，并不需要这么“笨重”的软件来实现挑选 Raw 格式图片的需求，毕竟我只想要提高效率而已，怎么方便怎么来。

既然有了需求和场景，那就自己来做工具好了。做不来？没关系，有 AI 嘛，只要把需求说清楚，这点小事还不是手到擒来的嘛！

用我下面的提示词，你也能自己制作一个小工具了，抛砖引玉，只要你有明确的想法，制作其它小工具也是不在话下的。打包成 exe 程序、发布 APP，这些都是可以的，打开思路，前面是一片广阔的天地！

 

## 二、参考提示词

在【文件夹】下，开发一个功能完整的 RAW 照片挑选 HTML 网页程序，具体需求如下：

1. 运行模式：
   - 支持窗口模式和全屏模式两种运行方式，提供清晰的切换控制
   - 窗口模式下支持标准窗口操作（最小化、最大化、关闭）

2. 界面设计：
   - 采用简洁、清新且具有科技感的 UI 设计风格
   - 确保界面布局合理，操作流程直观，视觉层次分明
   - 响应式设计，适应不同屏幕分辨率

3. 核心功能模块：

   a) 来源文件夹选择：
   - 提供醒目的"来源文件夹"按钮
   - 点击后打开系统文件夹选择对话框，允许用户指定本地文件夹
   - 验证所选文件夹的可访问性，并提供错误提示机制

   b) 照片预览与管理：
   - 成功选择来源文件夹后，自动扫描并显示该文件夹内所有JPG/JPEG格式照片的缩略图
   - 实现多种自定义排序功能：按文件名（升序/降序）、修改日期（最新/最早）、文件大小（从大到小/从小到大）
   - 缩略图显示应包含基本文件信息（文件名、修改日期、文件大小）

   c) 照片查看与操作：
   - 点击缩略图可进入全屏查看模式（保持原始长宽比）
   - 全屏模式提供以下功能按钮：
     * 放大功能（支持多级放大）
     * 缩小功能（支持多级缩小）
     * 旋转方向（顺时针/逆时针 90 度旋转）
   - 键盘快捷键支持：
     * 小键盘左箭头：切换到上一张照片
     * 小键盘右箭头：切换到下一张照片
     * 小键盘上箭头：勾选当前照片
     * 小键盘下箭头：取消勾选当前照片
   - 实现勾选状态的视觉指示（如边框高亮、勾选标记等）
   - 实时记录所有勾选过的 JPG / JPEG 照片名称

   d) 导出功能：
   - 提供"完成"按钮，点击后结束选择流程并切换到导出界面
   - 导出界面提供"导出文件夹"按钮
   - 支持选择现有文件夹或创建新文件夹作为导出目标
   - 实现文件复制功能：将来源文件夹中所有勾选的 JPG / JPEG 照片对应的同名 RAW 格式文件（包括但不限于 CR2、NEF、ARW、3FR、DNG、DATA、SR2 格式）复制到指定的导出文件夹
   - 提供文件复制进度指示和完成提示

4. 技术要求：
   - 确保应用程序在主流电脑操作系统上稳定运行
   - 处理大尺寸照片时保持界面响应流畅
   - 实现必要的错误处理和用户提示机制
   - 确保文件复制过程的安全性和完整性

5. 交付物：
   - 可直接运行的 HTML 文件
   - 基本的用户操作说明文档

应用程序应优先保证核心功能的稳定性和易用性，同时注重用户体验和界面美观度。
