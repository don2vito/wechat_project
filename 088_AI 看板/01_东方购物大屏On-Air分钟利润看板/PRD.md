# 产品需求文档（PRD）

> 项目名称：东方购物大屏 On-Air 分钟利润看板
> 版本：v1.0
> 状态：已确认

---

## 一、产品概述

东方购物大屏 On-Air 分钟利润看板是一个纯前端的交互式数据看板应用，用于展示和分析东方购物频道的 On-Air 播出分钟利润数据。用户上传当期和对比期两份 Excel 文件后，系统自动解析、清洗数据，并通过图表和表格动态呈现分钟利润和播出档数的对比分析结果。目标用户为东方购物运营团队，用于大屏展示和日常数据监控。

---

## 二、核心功能需求

### 功能1：数据文件上传

**描述：** 用户通过文件上传组件分别上传"当期"和"对比期"两份 Excel 文件（.xlsx/.xls 格式）。系统在浏览器端解析文件内容，提取所需字段。

**用户操作：**
1. 在页面上找到"当期文件"上传区域，点击或拖拽上传 Excel 文件
2. 在页面上找到"对比期文件"上传区域，点击或拖拽上传 Excel 文件
3. 两份文件均上传成功后，系统自动触发数据解析和处理流程

**预期结果：**
- 上传区域显示已上传文件名
- 系统自动解析 Excel 文件，提取数据行
- 解析完成后自动执行数据清洗和看板渲染
- 如果文件格式不正确或缺少必要列，显示错误提示

**Excel 必要列：**
| 列名 | 说明 |
|------|------|
| 播放日期 | 日期格式，如 2024-01-15 |
| 播放开始小时 | 整数，0-23 |
| 播放类型 | 文本，如"再播放"或其他 |
| md名 | 品类名称 |
| 频道 | 频道编号，如 11、12 |
| On-Air利润 | 数值，单位：元 |
| 希望播放（分） | 数值，单位：分钟 |

---

### 功能2：数据自动清洗与处理

**描述：** 系统在解析 Excel 后，自动执行一系列数据清洗规则，剔除无效数据，并计算衍生字段。

**用户操作：** 无需用户操作，系统自动执行。

**预期结果：** 清洗后的数据集用于后续图表和表格渲染。

**清洗规则（按顺序执行）：**

1. **剔除凌晨时段数据：** 移除"播放开始小时"为 1、2、3、4 的所有数据行
2. **剔除特定品类数据：** 移除"md名"为"全装修工程"、"寿险"、"汽车整车"、"房产"的所有数据行
3. **计算工作日/周末：** 根据"播放日期"判断该日期是工作日（周一至周五）还是周末（周六、周日），生成"分类"字段
4. **计算直录播类型：** "播放类型"为"再播放"的标记为"录播"，其余标记为"直播"，生成"直录播"字段
5. **计算时间段：** "播放开始小时"为 8-17 的标记为"白天"，18-23 的标记为"晚间"，生成"时间段"字段
6. **计算分钟利润：** 分钟利润 = On-Air利润 / 希望播放（分），生成"分钟利润"字段
7. **计算同比变化：** 同比变化 = ((当期分钟利润 - 对比期分钟利润) / 对比期分钟利润) * 100%，生成"同比变化"字段（百分比，保留2位小数）

---

### 功能3：整体分钟利润图表（不受筛选影响）

**描述：** 页面顶部展示一个 Chart.js 混合图表（柱状图 + 折线图），展示所有频道和全部频道的整体分钟利润情况。该图表不受任何筛选器影响，始终展示全量数据的聚合结果。

**用户操作：** 无需用户操作，数据上传并清洗完成后自动渲染。

**预期结果：**
- X 轴：频道分组（11频道、12频道、全部）
- 柱状图：当期分钟利润（蓝色柱子）
- 柱状图：对比期分钟利润（灰色柱子）
- 折线图：同比变化百分比（橙色折线，右侧 Y 轴）
- 图表有图例说明
- 数据标签显示在柱子顶部

**数据聚合逻辑：**
- 按频道分组，对清洗后的数据计算：总 On-Air利润 / 总希望播放（分）= 整体分钟利润
- "全部"组 = 所有频道数据汇总后的分钟利润
- 同比变化 = ((当期 - 对比期) / 对比期) * 100%

---

### 功能4：品类分钟利润表格（受筛选影响）

**描述：** 以表格形式展示各品类的分钟利润对比数据。表格数据受筛选器联动控制。

**用户操作：** 通过筛选器选择条件后，表格自动更新。

**预期结果：**
- 表格列：品类 | 当期分钟利润 | 对比期分钟利润 | 同比变化%
- 按当期分钟利润降序排列
- 同比变化列：正数显示绿色（带向上箭头），负数显示红色（带向下箭头），保留2位小数
- 当期分钟利润和对比期分钟利润保留2位小数，单位：元/分钟
- 表格支持横向滚动（品类较多时）

**数据聚合逻辑：**
- 根据当前筛选条件过滤数据
- 按品类（md名）分组
- 每个品类的分钟利润 = 该品类总 On-Air利润 / 该品类总希望播放（分）
- 同比变化 = ((当期 - 对比期) / 对比期) * 100%

---

### 功能5：播出档数表格（受筛选影响）

**描述：** 以表格形式展示各品类的播出档数对比数据。表格排序与区域2（品类分钟利润表格）保持一致。

**用户操作：** 通过筛选器选择条件后，表格自动更新。

**预期结果：**
- 表格列：品类 | 当期档数 | 对比期档数 | 对比（当期-对比期）
- 排序与品类分钟利润表格完全一致（按当期分钟利润降序）
- "对比"列：正数显示绿色，负数显示红色
- 当期档数和对比期档数为整数

**数据聚合逻辑：**
- 根据当前筛选条件过滤数据
- 按品类（md名）分组，统计每个品类的数据行数（即播出档数）
- 对比 = 当期档数 - 对比期档数

---

### 功能6：多维度筛选联动

**描述：** 提供 5 个筛选器，用户可组合使用。筛选器变化时，区域2（品类分钟利润表格）和区域3（播出档数表格）实时更新。区域1（整体分钟利润图表）不受筛选影响。

**用户操作：**
1. 点击"频道"下拉框，选择：全部 / 11频道 / 12频道
2. 点击"直录播"下拉框，选择：全部 / 直播 / 录播
3. 点击"时间段"下拉框，选择：全部 / 白天 / 晚间
4. 点击"分类"下拉框，选择：全部 / 工作日 / 周末
5. 点击"大分类"下拉框，选择：全部 + 14个动态品类（从上传数据中自动提取去重后的md名列表）

**预期结果：**
- 任意筛选器值变化后，区域2和区域3立即重新计算并渲染
- "大分类"下拉框的品类选项从当期和对比期数据合并后去重提取（排除已清洗的品类）
- 默认所有筛选器均为"全部"
- 筛选器之间为 AND 关系（交集）

**筛选逻辑：**
- 频道 = "全部"时不按频道过滤；选择具体频道时，只保留该频道的数据
- 直录播 = "全部"时不按直录播类型过滤；选择"直播"或"录播"时按对应字段过滤
- 时间段 = "全部"时不按时间段过滤；选择"白天"或"晚间"时按对应字段过滤
- 分类 = "全部"时不按工作日/周末过滤；选择"工作日"或"周末"时按对应字段过滤
- 大分类 = "全部"时不按品类过滤；选择具体品类时只保留该品类的数据

## 四、数据与存储

- **存储方式：** 纯前端应用，所有数据存储在浏览器内存中（JavaScript 变量），无需后端服务，无需数据库，无需持久化存储
- **数据处理位置：** 所有 Excel 解析、数据清洗、聚合计算均在浏览器端完成（使用 SheetJS 库）
- **数据生命周期：** 页面刷新后数据清空，用户需重新上传文件
- **隐私安全：** 数据不离开用户浏览器，不上传至任何服务器

---

## 五、边界条件与限制

1. **文件格式：** 仅支持 .xlsx 和 .xls 格式，不支持 .csv
2. **文件大小：** 建议单个文件不超过 10MB，过大文件可能导致浏览器卡顿
3. **必要列缺失：** 如果 Excel 中缺少必要列（播放日期、播放开始小时、播放类型、md名、频道、On-Air利润、希望播放（分）），应显示明确的错误提示，指出缺失的列名
4. **除零错误：** 当"希望播放（分）"为 0 时，分钟利润计算应跳过该行或标记为无效
5. **对比期数据为空：** 如果对比期某品类无数据，同比变化应显示为"N/A"或"--"
6. **筛选后无数据：** 如果筛选条件组合后无匹配数据，表格区域应显示"暂无数据"提示
7. **日期格式：** 支持 Excel 常见日期格式（Date 对象或日期序列号），需正确解析为 JavaScript Date
8. **频道值：** 假设频道列为数值类型（11、12），需处理可能的字符串类型

## 六、不在本版本范围内

1. 数据导出功能（导出为 Excel/PDF）
2. 用户登录和权限管理
3. 数据持久化存储（LocalStorage/数据库）
4. 历史数据对比（超过两个时期）
5. 自定义时间范围选择
6. 移动端适配（本版本仅针对大屏/桌面端优化）
7. 数据钻取（点击品类查看明细）
8. 实时数据推送
9. 多语言支持
10. 打印功能

---

## 七、开发工具与运行方式

- **开发方式：** 本项目为单个 HTML 文件，内嵌所有 CSS 和 JavaScript
- **运行方式：** 直接在浏览器中打开 HTML 文件即可运行，无需构建工具、无需安装依赖
- **外部依赖：** 通过 CDN 加载（SheetJS、Chart.js、GSAP、Lucide Icons、Google Fonts）
- **推荐浏览器：** Chrome 90+、Edge 90+、Firefox 88+、Safari 14+
- **开发工具建议：** 使用 Cursor 或 Trae 等 AI 编程工具直接生成完整代码
