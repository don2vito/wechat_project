# 交互式 On-Air 分钟利润看板实施计划

## 概要

基于现有白色主题 HTML 模板（`东方购物大屏-OnAir分钟利润看板.html`），改造为交互式数据看板应用。用户上传两个 Excel 文件（当期 + 对比期），系统自动解析、清洗数据，并按筛选条件动态渲染图表和表格。

## 当前状态分析

- **现有模板**：白色主题静态看板，包含标题栏、4个筛选器（硬编码选项）、CSS柱状图、两个静态数据表格
- **数据文件**：两个结构一致的 Excel 文件（70列，459行/453行），包含频道、播放日期、播放类型、大分类名称、On-Air利润、希望播放（分）等关键字段
- **大分类名称**：14个唯一值（个护美妆、保健护理、家具家饰、家用电器、家装建材、居家日用、旅游出行、流行服饰、消费服务、珠宝饰品、生活美食、艺术收藏、运动户外、鞋靴箱包）
- **播放类型**：4种（再播放、次新品播放、直播新商品、直播重拍播放），其中"再播放"为录播，其余为直播

## 数据处理规则

| # | 规则 | 实现方式 |
|---|------|---------|
| 1 | 剔除"播放开始小时"为1、2、3、4的数据 | `filter` + 数组包含判断 |
| 2 | 剔除"md名"为全装修工程、寿险、汽车整车、房产的数据 | `filter` + 排除列表 |
| 3 | 根据"播放日期"判断工作日/周末 | `new Date().getDay()`，0和6为周末 |
| 4 | "播放类型"为"再播放"的是录播，其它为直播 | 条件映射 |
| 5 | "播放开始小时"8-17为白天，18-23为晚间 | 条件映射 |
| 6 | 分钟利润 = On-Air利润 / 希望播放（分） | 行级计算 |
| 7 | 同比变化 = ((当期 - 对比期) / 对比期) * 100% | 聚合后计算 |

## 看板三大区域

### 区域1：整体分钟利润情况（不受筛选影响）
- 使用全部处理后数据（不经过筛选）
- Chart.js 混合图表：柱状图（当期vs对比期）+ 折线图（同比变化%）
- 按频道分组：11频道、12频道、全部
- 分钟利润 = 品类总On-Air利润 / 品类总希望播放分钟

### 区域2：品类分钟利润（受筛选影响）
- 表格：品类 | 当期分钟利润 | 对比期分钟利润 | 同比变化%
- 品类 = 大分类名称
- 按当期分钟利润降序排列

### 区域3：播出档数（受筛选影响）
- 表格：品类 | 当期档数 | 对比期档数 | 对比（当期-对比期）
- 档数 = 数据行数
- 品类排序跟随区域2

## 筛选选项

| 筛选项 | 选项列表 | 影响区域 |
|--------|---------|---------|
| 频道 | 全部频道、11频道、12频道 | 区域2、3 |
| 直/录播 | 全部、直播、录播 | 区域2、3 |
| 时间段 | 全部、白天、晚间 | 区域2、3 |
| 分类 | 全部、工作日、周末 | 区域2、3 |
| 大分类 | 全部大分类 + 14个动态品类 | 区域2、3 |

## 实施步骤

### 步骤1：HTML结构改造
- **文件**：`/workspace/东方购物大屏-OnAir分钟利润看板.html`
- 在 `<head>` 中新增 CDN：SheetJS (xlsx) 和 Chart.js
- 在 header 和 filter-bar 之间插入文件上传区域（两个上传框：当期、对比期）
- 筛选栏从4列改为5列，调整选项内容，添加 id 属性
- 图表区域标题改为"整体分钟利润情况"，内部替换为 `<canvas id="channelChart">`
- 左表标题改为"品类分钟利润"，右表标题改为"播出档数"
- 两个表格 `<tbody>` 添加 id（`profitTableBody`、`countTableBody`）
- 添加空状态占位提示

- `.filter-bar` grid 从 `repeat(4, 1fr)` 改为 `repeat(5, 1fr)`
- 新增表格滚动容器（`max-height` + `overflow-y: auto` + sticky表头）
- 新增空状态样式
- 调整响应式断点

### 步骤3：JavaScript — 数据层
- 全局状态对象 `state`（rawData、processedData、filters、categories、filesLoaded）
- `handleFileUpload()` — FileReader + XLSX.read + sheet_to_json
- `parseDate()` — 兼容 Excel 序列号和字符串格式
- `cleanData()` — 执行6条数据处理规则
- `applyFilters()` — 根据当前筛选条件过滤数据
- `aggregateByCategory()` — 按品类聚合分钟利润和档数

### 步骤4：JavaScript — 渲染层
- `renderAll()` — 总控函数，调用三个区域渲染

- `renderProfitTable()` — 区域2表格渲染（动态生成行）
- `renderCountTable()` — 区域3表格渲染（排序跟随区域2）
- 辅助函数：`formatNumber()`、`renderChangeBadge()`、`renderCountDiff()`

### 步骤5：JavaScript — 交互层

- 上传状态 UI 更新（文件名显示、边框变色）
- 适配现有 GSAP 动画

### 步骤6：验证
- 上传两个 Excel 文件，验证数据解析正确
- 验证6条数据处理规则执行正确
- 验证区域1不受筛选影响
- 验证区域2和3受筛选联动

## 关键技术决策

1. **CDN 依赖**：SheetJS（浏览器端解析Excel）、Chart.js（图表渲染），无需后端
2. **聚合方式**：品类分钟利润 = 品类总On-Air利润 / 品类总希望播放分钟（而非各行分钟利润的平均），避免时长权重偏差
3. **Excel日期处理**：`parseDate()` 兼容序列号和字符串两种格式
4. **空值处理**：某品类在对比期不存在时，对比期值显示0，同比变化显示 N/A
5. **表格滚动**：14个品类可能超出视口，添加 sticky 表头 + 滚动容器
